# 演练2-个人AI工具箱组合设计

## 演练概述

**时长**: 20分钟（15分钟案例实践 + 5分钟效果展示）
**目标**: 通过真实工作场景验证AI工具效果，掌握实际应用方法
**重点**: 案例验证 → 效果对比 → 经验总结 → 推广应用
**场景**: 电信行业团队成员AI工具快速体验

## ✅ 核心环节：案例验证实践（15分钟）

选择符合自己岗位的案例进行实践操作，体验AI工具在真实工作场景中的效果

### 📝 案例1：测试工程师 - 用例生成体验

#### 场景描述

为"话费充值功能"快速生成测试用例

#### 操作步骤

```
步骤1: 复制提示词到AI工具（1分钟）

步骤2: 使用以下提示词（2分钟）

提示词：
"为手机话费充值功能生成测试用例。
功能：用户输入手机号、充值金额（10/30/50/100元），选择支付方式完成充值。

请生成以下测试用例：
1. 正常充值流程（3个用例）
2. 异常场景（手机号错误、余额不足等，3个用例）
3. 边界测试（金额边界、号码格式等，2个用例）

每个用例包含：编号、用例名称、测试步骤、预期结果"

步骤3: 查看生成结果（2分钟）
观察AI生成的用例是否覆盖主要场景
```

#### 预期输出

- 快速生成8-10个测试用例
- 覆盖正常、异常、边界场景
- 体验AI辅助测试用例设计的效率
